# Sensor Tower《State of AI Report 2026》解讀報告（最終版）

**來源檔案：**
- `sensor_tower_report_draft.md`
- `sensor_tower_state_of_ai_2026_analysis.md`

**性質：** 這份 Sensor Tower 報告與每週 AI 新知週報不同。週報主要追蹤「廠商發布了什麼」：Claude Code、Codex、Antigravity、Cursor、local LLM、agent 架構等工具更新。Sensor Tower 報告則用 App、網站、廣告與零售流量數據，補上「使用者和市場實際怎麼變化」。

**閱讀原則：** 以下區分「報告/週報原文事實」與「Codex 推論/判斷」。若來源未提供精確日期或計算基準，會明確標註為需要澄清。

## 之前不知道的關鍵發現

### 1. ChatGPT 真實受眾市佔率跌破 50%，且有具體觸發事件

**報告原文事實：**
- ChatGPT 真實受眾市佔率在 2026-03 首次跌破 50%，2026-05 為 **46%**；Gemini 為 **28%**，Claude 為 **10%**。
- 報告整理的事件時間軸是：2/26 Anthropic 聲明拒絕與美國國防部合作特定監控應用，2/28 OpenAI 宣布與美國戰爭部合作協議，3/1~3/5 Claude 單日下載量首次超越 ChatGPT。
- ChatGPT 解除安裝峰值為 **+202%**，analysis.md 進一步寫明是 **3/9~3/15 那週**。

**討論點：** 我們之前完全沒追蹤過「企業政治立場 → 用戶解除安裝」這種因果鏈的具體幅度數據，這對任何 AI 公司的公關/政策決策都是一個可量化的風險係數。

**Codex 觀點：**

這裡最值得注意的不是「ChatGPT 是否失去領先」，而是使用者遷移速度。W20~W25 週報追蹤的是模型、CLI、agent workflow 的功能戰，沒有量化「用戶會因供應商政治/政策事件而卸載」這件事。Sensor Tower 的 +202% 提供了另一種風險指標：AI 工具不只比能力，也比信任與品牌風險。

不過，來源只支持「事件後解除安裝飆升並有部分用戶轉向 Claude」這個結論；不能把它寫成單一原因完全解釋 ChatGPT 市佔率下降。市佔率下降也可能同時受到 Gemini、Claude、DeepSeek、廣告化、留存變化等因素影響。

### 2. Claude 的商業化轉型幅度比想像中大

**報告原文事實：**
- Claude ARPU 從 2025-09 不到 **$0.5** 上升到 2026-05 的 **$2.76**。
- Claude 在美國的市佔率從 2025-12 的 **5%** 上升到 2026-05 的 **14%**。
- analysis.md 將 Claude 的 ARPU 成長連到「商業/研究/coding 工作流」成熟。

**週報原文事實：**
- W24 週報記錄：Anthropic 宣布自 **2026-06-15** 起，Claude Agent SDK 相關用量從訂閱池分離，移到獨立月費 credit pool，以全額 API 費率計費，無 rollover。
- W24 列出的受影響項目包含 Claude Agent SDK、`claude -p`、Claude Code GitHub Actions、第三方 Agent SDK 應用。
- W24 也寫明：原本訂閱隱性補貼了 **15~30x** API 費差，此次等同取消補貼。

**討論點：** 這個 ARPU 成長曲線是否能解釋 Anthropic 為何敢在同期推出 Agent SDK 計費分池？兩者時間點吻合，可能是同一個「轉向營收最大化」策略的兩個面向。

**Codex 觀點：**

我的判斷：**可以視為同一個大方向，但不能證明是同一個直接策略動作。**

證據較強的部分是方向一致。Sensor Tower 顯示 Claude 到 2026-05 已經把 ARPU 拉到 $2.76，W24 則在 2026-06-15 生效的 Agent SDK 分池中取消訂閱補貼、改用全額 API 費率。兩者都指向同一件事：Anthropic 正在把高價值、專業、coding、agentic workflow 用戶從「訂閱包吃到飽」推向更精準的用量變現。

證據較弱的部分是因果。ARPU 數據截止 2026-05，Agent SDK 分池生效日是 2026-06-15；時間上是 ARPU 已經上升在前，分池生效在後。因此更合理的說法是：**ARPU 上升證明 Claude 的專業用戶願意付費，這降低了 Anthropic 推出更嚴格計費分池的風險。** 但不能說 6/15 分池「造成」2026-05 的 ARPU 成長。

不確定性有三點：第一，來源沒有拆出 ARPU 成長來自訂閱升級、企業採購、API、Claude Code，還是其他 IAP 結構。第二，W24 的分池主要影響 Agent SDK、`claude -p`、GitHub Actions 等 programmatic usage，不等同於所有 Claude 消費者使用。第三，週報提到 IPO 前最大化 API 收益，但 Sensor Tower 報告本身沒有證明 IPO 動機。

結論：**同一個「從增長補貼轉向專業用戶變現」策略方向，證據中等；若說兩者是同一個內部策略專案，證據不足。**

### 3. Agentic Shopping 已有實證 ROI 數字，不只是概念

**報告原文事實：**
- Amazon Rufus 使用者轉換率 **>40%**，非使用者約 **~20%**；轉換前停留時間為 **40 分鐘**，非使用者為 **15 分鐘**。
- Walmart Sparky 在 2025-12~2026-04 的 MAU 滲透率 **+27%**、DAU **+50%**；Walmart 官方揭露 Sparky 用戶 AOV 高出非用戶 **~35%**。
- Amazon 封鎖 ChatGPT 網路爬蟲抓取商品目錄；Target、Walmart 反而宣布與 ChatGPT 官方合作。analysis.md 寫明 Walmart/Target 的 GenAI 推薦流量佔比已突破 **1.5%**，Amazon 停滯在 **~0.5%**。

**討論點：** 這代表「要不要讓 AI 爬蟲抓取你的網站」已經是零售業的策略分歧點，不單純是技術問題。

**Codex 觀點：**

這組數據把 agentic shopping 從「可能有用」推進到「已經有商業 ROI 指標」。Rufus 和 Sparky 的數字不是模型 benchmark，而是交易漏斗數據：轉換率、停留時間、DAU、AOV。對企業來說，這比模型能力排名更接近採購決策。

我會把零售商分成兩條路線：Amazon 選擇保護商品目錄與自家助理入口；Walmart/Target 選擇讓 ChatGPT 成為外部發現入口。Sensor Tower 的數字顯示，至少在 GenAI 推薦流量佔比上，合作路線暫時比封鎖路線更快取得外部流量。

但這不能直接外推成「所有網站都應開放 AI 爬蟲」。報告提供的是零售場景，且主要是美國/英國/加拿大精選零售商；不同產業的內容價值、轉換路徑與資料授權風險不同。比較穩健的結論是：**AI crawler policy 已經是商業分發策略，不只是 robots.txt 技術選項。**

### 4. AI 廣告化的速度遠超預期

**報告原文事實：**
- analysis.md 寫明：ChatGPT 廣告測試自 **2026-02** 起，至 5 月底，廣告數與曝光用戶數成長 **~7倍**。
- 草稿的待辦清單提到「ChatGPT 廣告測試 2/9 起」，但 analysis.md 正文只支持「2026-02 起」，未提供 2/9 的正文佐證。
- ChatGPT 廣告類別中，Shopping + Software 合計近 **半數**。
- ChatGPT 廣告地理策略是避開加州/德州/佛州大城市，轉攻中西部/南部/山區「未飽和市場」。

**週報原文事實：**
- W24 追蹤 Anthropic 2026-06-15 Agent SDK 計費分池。
- W24 明確指出此變更直接影響 `claude -p` 與 GitHub Actions 自動化任務，且取消原本訂閱對 API 費差的 15~30x 隱性補貼。

**討論點：** ChatGPT 廣告測試與 Agent SDK 計費分池，是否是 OpenAI/Anthropic 同步加速變現的兩個不同訊號？兩者相似與差異為何？

**Codex 觀點：**

我的判斷：**是同一個產業階段的兩個變現訊號，但不是同一種商業模式。**

相似點有三個。第一，時間接近：ChatGPT 廣告測試在 2026-02 到 2026-05 快速放大，Anthropic 分池在 2026-06-15 生效。第二，兩者都把原本由平台吸收的價值轉成明確營收：OpenAI 把注意力與分發入口變成廣告庫存，Anthropic 把高成本 agent/programmatic usage 變成全額 API 費率。第三，兩者都顯示 AI 助理從「拉新與習慣養成」進入「使用量與注意力貨幣化」階段。

差異也很大。OpenAI 廣告是偏消費端與雙邊市場：使用者看到廣告，廣告主付費，Shopping/Software 成為早期採用類別。Anthropic Agent SDK 分池是偏開發者與企業端：使用者或團隊為 programmatic agent 用量付費，核心是成本回收與高價值工作流定價。前者測試的是「AI 助理能不能成為廣告入口」，後者測試的是「agentic automation 能不能按用量收錢」。

來源未提供 OpenAI 與 Anthropic 有協調行動的證據，因此不能寫成「同步策略」。較準確的寫法是：**兩家公司在相近時間點，各自沿著自身用戶結構加速變現。OpenAI 變現注意力與分發，Anthropic 變現專業用量與 automation。**

### 5. 整體大盤成長已明顯減速，但「非 Generative AI」類別反而加速

**報告原文事實：**
- analysis.md 的整體市場規模寫明：全球 GenAI 下載在 2026 H1 估計為 **23億**，**+7% HoH**；成長明顯放緩，H2 2025 為 **+26%**，H1 2025 為 **+71%**。
- 全球 GenAI IAP 營收在 2026 H1 預估超過 **$4B**，**+36% HoH**。
- 含 AI 詞彙 App 的 IAP 營收預估達 **$12B**，其中 Generative AI App 佔 **~$4B**。
- 扣除 Generative AI App 後，AI 相關營收仍 YoY **+40%**，達 **~$8B**，主力來自 Multimedia & Design、Health & Wellness、Jobs & Education，以及短劇 App 帶動的 Movies & TV Shows。

**週報原文事實：**
- W20~W25 主要追蹤 AI coding assistant、local LLM、agent 架構與工具發布。
- W22 有一個相近但不同的觀察：Antigravity、Claude Code、Cursor「功能收斂」，轉換成本不低，繼續用熟悉工具最合理。
- 指定週報中沒有找到「AI 工具疲勞」這個明確表述。

**討論點：** 「AI 泡沫降溫」敘事 vs「AI 滲透其他品類加速」是否矛盾，還是兩個獨立現象？

**Codex 觀點：**

我的判斷：**不矛盾，是兩個不同層級的現象。**

「AI 泡沫降溫」比較像是在描述 standalone GenAI App 的增長斜率放緩：下載 HoH 從 H1 2025 的 +71%、H2 2025 的 +26%，降到 H1 2026 的 +7%。這表示單純再推出一個通用聊天或內容生成 App，新增下載紅利正在減少。

「AI 滲透其他品類加速」則是在描述 AI 從獨立 App 變成橫向功能層。扣除 Generative AI App 後，含 AI 詞彙 App 的 IAP 仍 YoY +40%、達 ~$8B，代表使用者不一定下載新的 AI App，但會在影音、健康、教育、工作、短劇等既有品類裡為 AI 功能付費。

這和 W20~W25 週報的工具觀察可以對上：週報不是在說 AI 沒用，而是在呈現工具選擇變複雜、功能收斂、切換成本上升。Sensor Tower 補上的市場數據是：即使通用 GenAI App 增長降溫，AI 作為功能仍在擴散。比較準確的結論是：**泡沫降溫發生在「通用 GenAI App 增長敘事」，滲透加速發生在「AI 作為功能被整合進垂直品類」；兩者可以同時成立。**

### 6. 亞洲市場首次出現下載量下滑（13 個季度連續成長後）

**報告原文事實：**
- 亞洲 GenAI App 下載在 Q1 2026 下滑 **-3.3%**，是 13 個季度連續成長後首次下滑。
- 中東同樣下滑。
- 拉美與歐洲領漲；巴西已超越中國成為下載量第三大市場。
- 北美為下載第四大區域但持續成長；北美在 Q1 2026 IAP 營收領先全球，為 **$730M**，歐洲 **$521M**，亞洲 **$330M**。

**討論點：** 這跟我們之前完全沒追蹤的「中國/印度 AI App 採用觸頂」現象有關，值得確認是飽和效應還是其他結構性原因（監管、本土替代品競爭等）。

**Codex 觀點：**

來源只足以支持「亞洲 Q1 2026 下載下滑」與「巴西超越中國成為第三大下載市場」這兩件事。它不足以直接證明中國或印度已經採用觸頂，也沒有在來源檔案中提供監管、本土替代品競爭、價格或通路分解。

因此我會把這個發現寫成風險提示，而不是結論：**亞洲下載下滑可能代表成熟市場飽和、區域供給轉向本土平台、或通用 GenAI App 下載紅利結束，但來源未提供精確歸因。**

對週報讀者來說，這提醒我們不要只用美國工具發布節奏理解全球 AI 採用。W20~W25 幾乎都在追蹤美國大廠與開發者工具；Sensor Tower 顯示區域市場可能已經分化，未來若評估工具或產品策略，應把「用戶在哪個市場、是否願意付費、是否受本土平台影響」納入。

## 數字一致性檢查

### 沒有明顯矛盾、但需要精確標註的地方

1. **ChatGPT 解除安裝 +202%**
   - 草稿寫「解除安裝量單週 +202%」。
   - analysis.md 表格寫「解除安裝峰值 +202%（3/9~3/15 那週）」；執行摘要則寫「解除安裝率飆升 +202%」。
   - 判斷：方向一致，但「解除安裝量」與「解除安裝率」用詞不同。最終版應寫成「解除安裝峰值 +202%，analysis.md 指向 3/9~3/15 那週」，並註明來源未提供基準週與分母。

2. **Claude ARPU 倍數**
   - 來源寫 2025-09 不到 $0.5 → 2026-05 $2.76。
   - 若用 $0.5 當上限計算，2.76 / 0.5 = 5.52，因此「5 倍以上」成立；因起點是「不到 $0.5」，實際倍數可能高於 5.52。
   - 判斷：沒有矛盾，但不能寫成精確 5.52 倍，因為起點不是精確 $0.50。

3. **ChatGPT 廣告測試起點**
   - analysis.md 正文支持「2026-02 起，至 5 月底成長 ~7 倍」。
   - 草稿待辦清單提到「2/9 起」，但正文沒有展開。
   - 判斷：最終版若要寫精確日期，應標註「草稿待辦提到 2/9；analysis.md 正文只提供 2026-02」。目前採用較保守的「2026-02 起」。

4. **廣告測試期間長度**
   - 草稿寫「僅 3 個月（2026-02~05）」。
   - 若從 2 月初到 5 月底，接近 4 個日曆月；若從 2/9 到 5 月某個未指明日期，才較接近 3 個月。
   - 判斷：來源未提供完整日期範圍，建議寫「2026-02 起至 5 月底」而非強調精確 3 個月。

5. **GenAI 成長率的口徑**
   - 草稿寫「GenAI App 下載 HoH 成長從 71%（H1 2025）降至 36%（H1 2026）」。
   - analysis.md 的市場規模段落寫：全球下載 H1 2026 為 23 億、+7% HoH；H2 2025 +26%、H1 2025 +71%。同段另寫全球 IAP 營收超過 $4B、+36% HoH。
   - 判斷：草稿疑似把「下載成長」與「IAP 營收成長」混在一起。較一致的寫法是：下載成長從 H1 2025 +71%、H2 2025 +26% 降到 H1 2026 +7%；IAP 營收在 H1 2026 為 +36% HoH。

6. **True Audience 市佔率與前三大使用時間佔比**
   - analysis.md 寫前三大 AI 助理 App 佔全球該品類使用時間 89%。
   - 同時寫 2026-05 True Audience：ChatGPT 46%、Gemini 28%、Claude 10%，三者合計 84%。
   - 判斷：不構成矛盾，因為一個是「使用時間佔比」，一個是「True Audience 市佔率」，口徑不同。

7. **Claude 10% vs 美國 14%**
   - analysis.md 寫 2026-05 Claude True Audience 為 10%，又寫 Claude 美國市佔率 2026-05 為 14%。
   - 判斷：不構成矛盾，因為一個是整體 True Audience，一個是美國市場。

## 對 D:\AI_study 週報讀者最有實際參考價值的數據

這裡的讀者關注 AI coding assistant、local LLM、agent 架構。因此最有價值的不是零售或廣告本身，而是能幫助判斷「該用哪個 AI 工具」的市場訊號。

### 1. Claude ARPU：2025-09 不到 $0.5 → 2026-05 $2.76

這是評估 Claude Code / Claude Agent SDK 的核心數據。它說明 Claude 正在從一般聊天工具轉向高付費專業工作流，而 W24 的 Agent SDK 分池進一步證實 programmatic usage 會被單獨計價。對實務選型的意義是：Claude 仍可能是高價值 coding/agent workflow 的強工具，但必須把 `claude -p`、GitHub Actions、Agent SDK 用量納入預算治理，不能再假設訂閱包吃到飽。

### 2. AI Assistant 市場集中但仍會遷移：前三大使用時間 89%，ChatGPT 46%、Gemini 28%、Claude 10%

這組數據支持「多工具策略」。ChatGPT 仍大，但不再是唯一入口；Claude 和 Gemini 有明確成長空間。對週報讀者來說，這意味著 coding assistant 不應只押單一供應商：Codex、Claude Code、Gemini/Antigravity、Cursor/Copilot 的能力與成本會快速變動，保留替代路線比追逐單一最佳工具更務實。

### 3. ChatGPT 廣告測試 2026-02 至 5 月底成長 ~7 倍

這不是 coding benchmark，但它影響工具體驗與商業模式。當通用 AI 助理開始廣告化，使用者要評估的不只是模型能力，還包括介面干擾、資料/注意力變現、企業政策限制。與 W24 的 Anthropic 計費分池放在一起看，可以得到一個實用判斷：**雲端 AI 工具正在全面進入變現階段；local LLM 不是必然更強，但會成為成本、隱私與可控性的避險選項。**

## 結論

Sensor Tower 報告補上的不是「又有哪些新模型」，而是市場已經如何投票：ChatGPT 仍領先但不再不可撼動，Claude 的專業付費能力正在被驗證，AI 助理已開始從補貼增長轉向廣告、API、agent 用量等多路變現。

對 W20~W25 週報脈絡來說，最重要的交叉結論是：我們追蹤的 coding assistant、local LLM、agent 架構不是孤立工具更新，而是在同一個商業轉折中競爭。未來評估「該用哪個 AI 工具」時，應同時看三件事：能力是否足夠、工作流是否可控、長期成本是否會因平台變現策略而改變。
