### 1. Claude Sonnet 5 發佈：最具 Agentic 能力的 Sonnet，接近 Opus 4.8（2026-06-30）

Anthropic 在 **2026-06-30** 正式推出 **Claude Sonnet 5**，定位為「史上最具 agentic 能力的 Sonnet」——計畫、工具呼叫、自主執行能力大幅躍進，且僅需 Sonnet 等級的定價。

**技術規格：**
- **上下文窗口：1M tokens**（與 Opus 4.8 相同）
- **最大輸出：128K tokens**
- **Token 計算變化：** 相同輸入文字產生的 token 數約比 Sonnet 4.6 多 **30%**（需注意用量預算）
- 工具集與平台功能與 Sonnet 4.6 相同，但 agentic 執行穩定性大幅改善

**定價：**
| 項目 | 優惠期（至 2026-08-31） | 標準價（9 月起） |
|------|------------------------|-----------------|
| Input | $2 / 1M tokens | $3 / 1M tokens |
| Output | $10 / 1M tokens | $15 / 1M tokens |

**能力提升亮點（Benchmark）：**
- **SWE-bench Pro：63.2%**（Opus 4.8：69.2%）
- **OSWorld-Verified：81.2%**（Opus 4.8：83.4%）
- **Terminal-Bench 2.1：80.4%**（實際超越 Opus 4.8 的 74.6%！）
- **BrowseComp agentic search：84.7%**
- 原本需要更大更貴模型才能完成的自主任務，Sonnet 5 均可處理

**可用性：** 即日起 Free / Pro / Max / Team / Enterprise 全計畫預設模型（Free 和 Pro 計畫直接升級為預設）

**對你的意義：** Claude Code 的底層已自動升級至 Sonnet 5。若你有注意到本週 Claude Code 在多步驟任務中表現更流暢，這就是原因。token 多 30% 的注意點：如果你有設定 token 預算上限，可能需要調整。

### 2. Claude Fable 5 + Mythos 5 出口管制解除，全球恢復存取（2026-06-30 → 07-01）

本週最具戲劇性的進展：**Claude Fable 5 和 Mythos 5 的出口管制在 18 天後正式解除**。

**事件回顧（W25/W26 持續追蹤）：**
- 2026-06-09：Anthropic 推出 Fable 5 和 Mythos 5
- 2026-06-12：Amazon 研究人員發現 Fable 5 的 jailbreak——一組 prompt 可讓模型繞過安全規則、揭露軟體漏洞，甚至生成漏洞利用程式碼
- 2026-06-12：美國商務部發出緊急出口管制令，要求 Anthropic 暫停「所有外籍人士的存取，包含 Anthropic 外籍員工」
- **2026-06-30**：商務部正式解除出口管制
- **2026-07-01**：Fable 5 和 Mythos 5 重新上線，適用平台：claude.ai、Claude Platform API、Claude Code、Claude Cowork

**為何重要：**
- 首次有 AI 模型因「jailbreak 觸發安全疑慮」被政府緊急停用的案例，為 AI 法規史上的里程碑
- 管制解除意味商務部與 Anthropic 完成了某種技術或流程上的驗證，具體條件未公開
- Fable 5 和 Mythos 5 是目前 Anthropic 最高階模型（Fable 5 > Opus 4.8），恢復存取後可再次測試

**Anthropic 的解法（新 Cybersecurity Classifier）：**
- 為解除管制，Anthropic 訓練了一個針對該 jailbreak 技術的新安全分類器
- 分類器攔截率：**99%+ 以上**攔截該 jailbreak 手法
- **代價（重要）**：更多無害的 coding / debugging 請求也會被誤攔截，開發者使用時可能遇到比管制前更多的拒絕或迴避回應

**對你的意義：** 若你此前計畫試用 Fable 5，現在可以重新存取。但需留意：新 classifier 可能讓 Fable 5 在某些程式碼分析任務上比管制前更謹慎——若遇到意外拒絕，這是 trade-off 的一部分，不是 bug。

### 3. OpenAI GPT-5.6 Sol / Terra / Luna 三層模型限量預覽（2026-06-26）

OpenAI 在 **2026-06-26** 預告下一代模型系列 **GPT-5.6**，採三層架構：

| 模型 | 定位 | 定價（Input / Output） | 特點 |
|------|------|----------------------|------|
| **Sol** | 旗艦 | $5 / $30 per 1M tokens | 最強，agentic 能力大幅強化，coding / biology / cybersecurity |
| **Terra** | 平衡 | $2.50 / $15 per 1M tokens | GPT-5.5 同水準但便宜 2x |
| **Luna** | 快速低成本 | $1 / $6 per 1M tokens | 最快、最便宜 |

**新能力：**
- 新增 **max 推理強度**（比現有 extended thinking 更深）

**存取限制（目前）：**
- 僅限約 **20 家政府審批的合作機構**，透過 API 和 Codex 存取
- 一般 ChatGPT 用戶暫無法使用
- OpenAI 預計數週內逐步擴大開放

**對比 Claude Sonnet 5 定價：** GPT-5.6 Terra（$2.50/$15）與 Sonnet 5 優惠價（$2/$10）、標準價（$3/$15）處於同一競爭區間，GA 後兩者將直接競爭中端市場。

### 4. Claude apps gateway：企業版 Claude Code 的 SSO + RBAC 控制平面（2026-07-02）

Anthropic 在 **2026-07-02** 推出 **Claude apps gateway**，針對在 Amazon Bedrock 和 Google Cloud 上部署 Claude Code 的企業用戶。

**解決的問題：**
企業在大規模部署 Claude Code 時面臨「每個開發者需手動配置 API 憑證」、「無法集中控管用量和成本歸因」的痛點。

**架構亮點：**
- **自架控制平面**：以單一無狀態容器（需 PostgreSQL）部署在企業自己的 Linux 環境，**inference 流量不經 Anthropic**（Bedrock / Google Cloud 路由）
- **企業 SSO**：相容 Google Workspace、Microsoft Entra ID、Okta 等 OIDC 標準
- **RBAC**：組織 / 群組 / 個人三層的每日 / 每週 / 每月用量上限
- **人員管理**：onboarding = 加入 IdP，offboarding = 從 IdP 移除，無需手動管理 API key
- **開放協定**：Anthropic 發布 gateway 協定規格，允許第三方廠商建置相容實作

**對你的意義：** 個人用戶不影響，這是 B2B 企業功能。若你所在組織評估大規模 Claude Code 部署，這個 gateway 可讓 IT 集中管控、資料不出私有雲。

### 5. Colorado AI 法案修正：廢除歧視責任框架，改為 ADMT 揭露制度（2026-06-30 生效前被替換）

**SB24-205**（原科羅拉多 AI 法，美國第一部綜合州 AI 法，原訂 **2026-06-30** 生效）在生效日前就被立法機關廢除並替換：

**新法 SB26-189（2026-05-14 签署）：**
- 廢除原法的「演算法歧視禁止 + 注意義務（duty of care）」框架
- 改為管制「**自動化決策技術（ADMT）**在重大決策中的使用」
- 不再要求企業主動稽核偏見，改為揭露義務（說明「哪裡用了 AI 決策」）
- **新生效日：2027-01-01**

**背景：** 原法案受到白宮批評——認為仿效 EU AI Act 的「duty of care」框架過於嚴苛，恐阻礙美國 AI 競爭力。新法靠攏當前行政當局「以創新為優先」路線。

**對開發者的意義：** 若你的產品在 Colorado 用戶中使用 AI 做重大決策（如信用核准、職位申請篩選），留意 2027-01-01 起的 ADMT 揭露要求。相比原法案，合規門檻大幅降低。
