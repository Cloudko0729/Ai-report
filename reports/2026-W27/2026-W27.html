1. Claude Sonnet 5 發佈：最具 Agentic 能力的 Sonnet，接近 Opus 4.8（2026-06-30）

Anthropic 在 2026-06-30 推出 Claude Sonnet 5，定位為「史上最具 agentic 能力的 Sonnet」——計畫、工具呼叫、自主執行能力大幅躍進，且僅需 Sonnet 等級的定價。

技術規格：

  • 上下文窗口：1M tokens（與 Opus 4.8 相同）
  • 最大輸出：128K tokens
  • Token 計算：相同輸入文字產生的 token 數約比 Sonnet 4.6 多 30%（需注意用量預算）
  • 工具集與平台功能與 Sonnet 4.6 相同，但 agentic 執行穩定性大幅改善

能力提升亮點：

  • Agentic 規劃：接近 Opus 4.8，原本需要更大更貴模型才能完成的自主任務現在 Sonnet 5 就能跑
  • Coding 全生命週期：從需求解析、實作、測試到 PR 均有改善
  • 推理與知識工作：複雜多步驟推理明顯提升

可用性： 即日起 Free / Pro / Max / Team / Enterprise 全計畫預設模型

2. Claude Fable 5 + Mythos 5 出口管制解除，全球恢復存取（2026-06-30 → 07-01）

本週最具戲劇性的進展：Claude Fable 5 和 Mythos 5 的出口管制在 18 天後正式解除。

事件時間軸（W25/W26 持續追蹤）：

  • 2026-06-09：Anthropic 推出 Fable 5 和 Mythos 5
  • 2026-06-12：Amazon 研究人員發現 Fable 5 jailbreak，可讓模型繞過安全規則、揭露並生成漏洞利用程式碼
  • 2026-06-12：美國商務部發緊急出口管制令，要求暫停「所有外籍人士的存取，包含 Anthropic 外籍員工」
  • 2026-06-30：商務部正式解除出口管制
  • 2026-07-01：Fable 5 + Mythos 5 重新上線（claude.ai、Claude Platform API、Claude Code、Claude Cowork）

為何重要：

  • 首個「AI 模型因 jailbreak 觸發安全疑慮」被政府緊急停用的案例，是 AI 法規史上的里程碑
  • 管制解除意味商務部與 Anthropic 完成了某種技術或流程驗證，具體條件未公開
  • Fable 5 是目前 Anthropic 最高階模型（Fable 5 > Opus 4.8），現在再次可用

Anthropic 的解法（新 Cybersecurity Classifier）：

  • Anthropic 訓練了針對該 jailbreak 技術的新安全分類器，攔截率 99%+
  • 代價：更多無害的 coding / debugging 請求也可能被誤攔截——開發者使用 Fable 5 時可能遇到比管制前更多的拒絕回應
若你此前計畫試用 Fable 5，現在可以重新存取。留意新 classifier 可能讓 Fable 5 在某些程式碼分析任務上比管制前更謹慎——若遇到意外拒絕，這是 trade-off 的一部分，不是 bug。Fable 5 需要 Max 或更高計畫。

4. Claude apps gateway：企業版 Claude Code 的 SSO + RBAC 控制平面（2026-07-02）

Anthropic 在 2026-07-02 推出 Claude apps gateway，針對在 Amazon Bedrock 和 Google Cloud 上部署 Claude Code 的企業用戶。

解決的問題： 企業大規模部署 Claude Code 時，每個開發者需手動配置 API 憑證、無法集中管控用量和成本歸因。

架構亮點：

  • 自架控制平面：單一無狀態容器（需 PostgreSQL），inference 流量不經 Anthropic（Bedrock / Google Cloud 路由），資料留在私有雲
  • 企業 SSO：相容 Google Workspace、Microsoft Entra ID、Okta 等 OIDC 標準
  • RBAC：組織 / 群組 / 個人三層的每日 / 每週 / 每月用量上限
  • 人員管理：onboarding = 加入 IdP；offboarding = 從 IdP 移除，無需手動管理 API key
  • 開放協定：Anthropic 公開 gateway 協定規格，允許第三方（如 Cursor、Windsurf）建置相容實作
個人用戶不影響，這是 B2B 企業功能。若你所在組織評估大規模 Claude Code 部署，這個 gateway 讓 IT 集中管控、資料不出私有雲。

5. Colorado AI 法案修正：歧視責任框架廢除，改為 ADMT 揭露制度（生效延至 2027-01-01）

SB24-205（美國第一部綜合州 AI 法，原訂 2026-06-30 生效）在生效日前就被立法機關廢除並替換：

  • 新法 SB26-189（2026-05-14 簽署）：廢除「演算法歧視禁止 + 注意義務（duty of care）」框架
  • 改為管制「自動化決策技術（ADMT）在重大決策中的使用」——揭露義務取代主動稽核義務
  • 新生效日：2027-01-01
  • 背景：原法案受白宮批評仿效 EU AI Act 過於嚴苛，恐阻礙美國 AI 競爭力
若你的產品在 Colorado 用戶中使用 AI 做重大決策（如信用核准、職位申請篩選），留意 2027-01-01 起的 ADMT 揭露要求。合規門檻比原法案大幅降低。
