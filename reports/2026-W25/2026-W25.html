1. 🚨 Claude Fable 5 / Claude Mythos 5：上線 3 天即遭美國政府出口管制下架（截至今日仍未恢復）

Anthropic 於 2026-06-09 發布其首款 Mythos 等級模型 Claude Fable 5（公開版）與 Claude Mythos 5（內部/企業版），號稱效能全面超越 Opus 4.8，在程式碼撰寫與物理模擬上展現「一發命中」（one-shot）能力：

  • Stripe 實測：5000 萬行 Ruby 程式碼庫遷移，原本需數月工程量，一天內完成
  • 開發者單一提示詞 7 分鐘內生成具物理引擎的 3D 魔術方塊、2-3 分鐘建立雙擺物理模擬

但僅 3 天後（2026-06-12），美國政府以國家安全為由發出出口管制指令，要求 Anthropic 立即停止所有外國籍人士（包含 Anthropic 內部外籍員工）存取 Claude Fable 5 / Claude Mythos 5。由於無法即時區分使用者國籍，Anthropic 對全體用戶關閉了這兩個模型。政府關切的核心是模型可能被「越獄」用於繞過網路安全防護（如自動找漏洞）。

目前狀態（截至 2026-06-19）：尚未恢復，Anthropic 表示正在處理，但沒有公布恢復時間。Opus 4.8、Sonnet 4.6 等其他模型不受影響。

若你在 Claude Fable 5 上線當週試用過，目前無法再次存取；NotebookLM 簡報中提及的原按量計費資訊已因下架事件而失去時效性，暫不用理會。建議持續關注 Opus 4.8 作為當前最高可用層級。

2. Claude Managed Agents：排程部署（cron）+ Credential Vault 進入公開 Beta（2026-06-09）

Code with Claude Tokyo 活動上，Anthropic 同步宣布兩項 Claude Managed Agents 新能力，與 W24 報導的 Claude Code Dynamic Workflows 正式 GA 一起落地：

排程部署（Scheduled Deployments）：

  • 給 agent 設定 cron 排程，每次觸發自動啟動新 session 並完成任務，不需要自建 scheduler
  • 適合：每晚資料同步、每週合規掃描、每日摘要報告（Rakuten 已用於週/月報表自動化）
  • 可隨時暫停、恢復、封存部署，或手動觸發額外執行
  • 組織上限：1,000 個排程部署

Credential Vault：

  • API 金鑰等密鑰以佔位符形式存在 agent sandbox 中，真實金鑰只在請求白名單網域時於網路邊界注入
  • 即使 agent 被 prompt injection 操縱，模型 context 內永遠看不到真實密鑰

API 要求： 需在 API 呼叫加上 managed-agents-2026-04-01 beta header（API 版本字串，非事件日期）。

這項功能直接對應你週報系統「定期跑 nb.py 流程」的痛點——理論上可以把整套 WebSearch→NotebookLM→QA→HTML 流程包成排程 agent，每週自動觸發，不需手動說「開始本週報告」。值得列入下週深入研究清單。

3. Codex CLI 每週可嘗試功能（新增追蹤項目）

呼應你的要求，這裡列出 1-3 項本週可實際嘗試的 Codex 新功能：

  • codex sandbox setup --elevated（Windows alpha） — Windows 管理員可用此指令設定 sandbox 的提權配置。直接對應你目前遇到的痛點：codex:codex-rescue 子代理目前無法寫入 D:\AI_study\weekly_notes\（sandbox 不含 D 槽），這個新指令可能是解法，值得本週試一次。
  • Session 封存（/archive、codex archive / codex unarchive） — 把已完成的 QA rescue 對話封存，避免舊 thread 一直出現在 resume 候選清單中；封存後的 session 會被保護，不會被誤 resume。
  • /import 從 Claude Code 匯入設定 — 可選擇性匯入 Claude Code 的專案設定、setup 與近期對話到 Codex，若未來想讓 Codex 更了解這個週報專案的既有規則，可以試試。

4. 本地模型動態：MiniMax M3、Qwen 3.6 27B、Ollama v0.30.8

  • Ollama v0.30.8（2026-06-12）：擴大 GGUF 硬體支援、升級 Apple Silicon MLX engine
  • MiniMax M3：開源權重、1M token context + 原生視覺
  • Qwen 3.6 27B：SWE-bench 77.2%，適合本地 coding agent；本週 YouTube 實測顯示單張 RTX 4090 提示處理速度可達 3607 tps，但 Q4 量化版在 tool calling 上仍不穩定，建議用 BF16 全精度或 vLLM
  • NVIDIA Nemotron 3 Ultra、DeepSeek V4 Pro 同步上架 Ollama library
若要在本地跑 agent 自動化（如取代部分週報流程），Qwen 3.6 27B 配雙 4090 是目前性價比選擇，但 tool calling 穩定性仍是瓶頸，不建議完全取代雲端模型。
