## 本週最值得注意

### 1. Claude Code Opus 4.8 + Dynamic Workflows — 百agent並行正式可用

Anthropic 於 2026-05-28 發布 **Opus 4.8**，距 Opus 4.7 僅 41 天，是史上最短 Opus 迭代週期。核心亮點是 Claude Code 同步推出的 **Dynamic Workflows**（研究預覽）：

**Dynamic Workflows 運作方式：**
- Claude 在單一 session 內自動撰寫 orchestration script，動態啟動 **最多 1,000 個 subagent**
- 各 subagent 從獨立角度攻克問題，另有對抗性 agent 嘗試推翻其他 agent 的結論
- 答案收斂後才回報，支援跨越數十萬行程式碼的 codebase 完整遷移（從啟動到 merge）
- 已有回報：原本需要「以季計劃」的工程任務，數天內完成

**Opus 4.8 其他升級：**
| 項目 | Opus 4.7 | Opus 4.8 |
|------|---------|---------|
| SWE-bench Pro | 64.3% | **69.2%** |
| USAMO 2026（數學） | 69.3% | **96.7%** |
| 程式碼缺陷漏報率 | 基準 | **降低 4x** |
| Fast mode 成本 | 高 | **2x 標準費率（2.5x 速度）** |

**定價不變：** $5/1M input、$25/1M output（標準）；Fast mode $10/$50。

**其他 Claude Code 同步更新：**
- `.claude/skills` 目錄內的 Plugin 現在**自動載入**，無需 Marketplace 手動配置
- `/effort xhigh` 新指令：針對最難任務強制高推理負荷
- Auto mode 上線 Bedrock、Vertex、Foundry（Opus 4.7 和 4.8）

**對你的意義：** Dynamic Workflows 直接影響你的 Claude Code 使用方式——多個平行 subagent 可以同時研究、撰寫、驗證，與你現在的週報 multi-agent 流程概念完全契合。`.claude/skills` 自動載入讓自訂技能設定更簡便。

### 2. OpenAI Codex — Goal Mode GA + Windows Computer Use

本週 Codex 兩個重大功能同步推進：

**Goal Mode（2026-05-21 GA）**
- 從 beta 升為 **正式版**，在 Codex App、IDE Extension、CLI 全面可用
- 設定 `/goal`：定義目標與成功條件，Codex 自主推進**數小時甚至數天**，處理子任務、撰寫程式碼、迭代修正，無需持續提示
- 跨 session 持久化：budget reset 後目標不丟失

**Windows Computer Use（2026-05-29）**
- Codex 現在可在 **Windows 桌面**直接操控應用程式：看、點擊、輸入
- **前景模式**：Codex 在前台工作時，使用者需讓出螢幕（與 macOS 版相同限制）
- **Mobile Remote Control**：從 iPhone / Android 上的 ChatGPT 啟動 Windows 上的 Codex 任務、遠端監控進度
- 限制：歐盟、英國、瑞士目前不可用

**Appshots（macOS，同期更新）**
- 快捷鍵捕捉任何前景視窗（截圖 + 擷取文字），直接注入當前 Codex thread 作為 live context

**對你的意義：** Windows Computer Use 對你最直接——Codex 可以操作你 Windows 11 上的 GUI 應用（如瀏覽器、Obsidian、本機服務），不再局限於 terminal/file 操作。Goal Mode 搭配 Dynamic Workflows 的對比值得深思：前者是長期自主跑，後者是短期密集並行。

### 3. Microsoft Build 2026 — Project Polaris 宣告 GitHub Copilot 獨立

微軟在 Build 2026（2026-06-02~03）發布 **Project Polaris**，這是微軟自研的 coding 專用 MoE 大模型：

- **架構：** Mixture-of-Experts，含針對不同語言/框架的專屬 sub-module；推理時加入 chain-of-thought + tree-of-thought search
- **目標：** 從 **2026 年 8 月**起取代 GPT-4 Turbo，成為 GitHub Copilot 預設模型（現有用戶可選三個月 fallback 期）
- **基準：** 在 HumanEval、MBPP 超越 GPT-4 Turbo；Rust、Haskell 等低資源語言尤其明顯
- **基礎設施：** 跑在微軟自研 **Maia AI accelerator** 上，降低推理延遲與成本
- **Pro 功能：** 多檔案上下文最高 100,000 行 + 自主測試生成

同場發布的 **MAI 模型套件：**
- MAI-Transcribe-1（語音辨識）
- MAI-Voice-1、MAI-Image-2 首次開放商業使用

**對你的意義：** 若有在用 GitHub Copilot，8 月前留意遷移通知；Polaris MoE 在 Rust/Haskell 等語言的提升值得關注，但日常 Python/JS 任務優勢不確定。微軟與 OpenAI 的解耦訊號進一步強化——Copilot 未來可能不再是「GPT 的前端」。

## 依你背景的分類

### AI Coding Assistant
- **Dynamic Workflows + Goal Mode** 標誌著 agentic coding 進入「設定目標、放手讓跑」新階段，兩者路線不同：Dynamic Workflows 是爆發式並行（百 agent 短時間），Goal Mode 是持久型追蹤（數日自主）
- **Project Polaris** 代表 GitHub Copilot 徹底與 OpenAI 解耦，下半年 Copilot 使用者體驗可能出現較大變化

### Local LLM
- **Kimi K2.6**（MIT 授權）是目前最強的可商用本地 coding 模型，42B 活躍參數在多數消費級 GPU 可跑；`ollama pull kimi-k2.6` 可試
- **Qwen 3.6 27B** + MTP：若用 Ollama v0.24.0，可獲得約 2x 生成速度提升

### Agent 架構
- **Dynamic Workflows** 是目前最完整的 agentic orchestration 實作：自動生成 orchestration script → 平行 subagent → 對抗驗證 → 收斂回報。整個流程不需要用戶手動設計 DAG
- **Goal Mode** 的 `/goal` 機制值得研究：persistent goal across sessions = 一種輕量狀態管理方案
