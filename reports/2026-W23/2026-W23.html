1. Claude Code Opus 4.8 + Dynamic Workflows — 百 agent 並行正式可用（2026-05-28）

Anthropic 於 2026-05-28 發布 Opus 4.8，距 Opus 4.7 僅 41 天，是史上最短 Opus 迭代週期。核心亮點是 Claude Code 同步推出的 Dynamic Workflows（研究預覽）：Claude 在單一 session 內自動撰寫 orchestration script，動態啟動最多 1,000 個 subagent（同時最多 16 個並行），各 subagent 從獨立角度攻克問題，答案收斂後才回報。

實際案例： 將 Bun 專案從 Zig 移植到 Rust，75 萬行程式碼、99.8% 測試通過、首次 commit 到合併僅耗時 11 天。計畫存於 JavaScript 腳本變數中，不佔用對話 context window。

定價不變： $5/1M input、$25/1M output（標準）；Fast mode $10/$50。

Dynamic Workflows 直接影響你的 Claude Code 使用方式——多個平行 subagent 可同時研究、撰寫、驗證，與週報 multi-agent 流程概念完全契合。.claude/skills 自動載入讓自訂技能設定更簡便。

2. OpenAI Codex — Goal Mode GA + Windows Computer Use（2026-05-21~29）

Goal Mode（2026-05-21 GA）： 從 beta 升為正式版，在 Codex App、IDE Extension、CLI 全面可用。設定 /goal 定義目標與成功條件，Codex 自主推進數小時甚至數天，處理子任務、撰寫程式碼、迭代修正，無需持續提示；跨 session 持久化，budget reset 後目標不丟失。

Windows Computer Use（2026-05-29）： Codex 現在可在 Windows 桌面直接操控應用程式：看、點擊、輸入。前景模式下使用者需讓出螢幕，支援 Mobile Remote Control（從 iPhone / Android 啟動 Windows 上的任務並遠端監控）。限制：歐盟、英國、瑞士目前不可用。

Appshots（macOS 同期）： 快捷鍵捕捉任何前景視窗（截圖 + 擷取文字），直接注入當前 Codex thread 作為 live context。

Windows Computer Use 對你最直接——Codex 可操作 Windows 11 上的 GUI 應用（瀏覽器、Obsidian、本機服務），不再局限於 terminal/file 操作。Goal Mode vs Dynamic Workflows：前者長期自主跑，後者短期密集並行，路線不同值得對比。

3. Microsoft Build 2026 — Project Polaris 宣告 GitHub Copilot 獨立（2026-06-02~03）

微軟發布 Project Polaris，自研 coding 專用 MoE 大模型，從 2026 年 8 月起取代 GPT-4 Turbo 成為 GitHub Copilot 預設模型（現有用戶可選三個月 fallback 期）。架構採 Mixture-of-Experts，含針對不同語言/框架的專屬 sub-module；推理時加入 chain-of-thought + tree-of-thought search；在 HumanEval、MBPP 超越 GPT-4 Turbo，Rust、Haskell 等低資源語言尤其明顯。跑在微軟自研 Maia AI accelerator 上。

同場發布：

  • Windows Agent Framework (WAF)：MIT 授權開源，YAML 定義可跨本地、雲端、邊緣無縫遷移的 agent
  • DirectML 2.0：抽象化 Intel/AMD/Qualcomm NPU 差異，啟用本地 AI 推理
  • WSL 3：提升 Linux 工具鏈在 Windows 上的速度
  • MAI 模型套件：MAI-Transcribe-1、MAI-Voice-1、MAI-Image-2 首次開放商業使用
若有在用 GitHub Copilot，8 月前留意遷移通知；Polaris MoE 在 Rust/Haskell 的提升值得關注。WAF 的 MIT 開源 YAML 規範值得研究，避免重造 agent 基礎設施輪子。微軟與 OpenAI 解耦訊號強化——Copilot 未來可能不再是「GPT 的前端」。
