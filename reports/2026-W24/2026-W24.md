### 1. 🚨 Anthropic June 15 計費變更 — Agent SDK 強制分池（直接影響 Claude Code 用法）

Anthropic 宣布自 **2026-06-15** 起，Agent SDK 相關用量從訂閱池分離，移至獨立的月費 credit pool，以 **全額 API 費率**計費，無 rollover：

**受影響（新 credit pool）：**
- Claude Agent SDK
- `claude -p`（headless/programmatic CLI）
- Claude Code GitHub Actions
- 第三方 Agent SDK 應用

**不受影響（維持訂閱池）：**
- Claude.ai web / desktop / mobile 對話
- Claude Code 在 terminal 互動式使用
- Claude Cowork

**各方案 credit 額度：**
| 方案 | 月 credit | 約可用量估算 |
|------|-----------|------------|
| Pro | $20 | ~4M tokens（Sonnet 級） |
| Max 5x | $100 | ~20M tokens |
| Max 20x | $200 | ~40M tokens |

**重要細節：** credit 按人頭計（不共池）、月底清零不滾入、用完後自動停擋（除非手動開啟 overflow billing）。原本訂閱隱性補貼了 **15~30x** API 費差，此次等同取消補貼。

**立即行動：** 在 6/15 前登入 Claude 帳號確認 credit 申領信件，並決定是否開啟 overflow billing。

**對你的意義：** `claude -p` 和 GitHub Actions 自動化任務會消耗 credit；若你有跑 Agent SDK 腳本，本週確認額度後制定預算。如每月 `claude -p` 用量超過 $20 等值，需開啟 overflow 或升級 Max 方案。

### 2. Claude Code：Dynamic Workflows 預設開啟 + Ultracode 新設定（2026-06-02~03）

兩個重要 Claude Code 行為更新同步落地：

**Dynamic Workflows 預設狀態變更：**
- **Max 和 Team 方案：** Dynamic Workflows 現在**預設開啟**，無需任何設定
- **Enterprise：** 預設關閉，管理員可在 Claude Code settings 啟用
- **Pro：** 需手動在 `/config → Dynamic Workflows` 啟用

**新設定：`/effort ultracode`**
- 結合 `xhigh` 推理力度 + 自動判斷何時啟動 workflow
- Claude 自動評估任務複雜度，決定是否啟動 Dynamic Workflows
- 適合：大型重構、跨模組修改、複雜 bug 追蹤
- 使用方式：session 開始時輸入 `/effort ultracode`

**其他更新（release notes）：**
- Fallback models 支援（主模型失敗時自動切換）
- 更廣的 deny-rule glob 規則
- 跨 session 訊息安全性強化
- 20+ 法律 MCP connector + 12 個 practice-area plugin（企業向）

**Managed Agents 升級：**
- 現在可在**自管 sandbox**中運作（Cloudflare、Daytona、Modal、Vercel 等）
- 連接**私有 MCP server**（企業 internal tools）

**對你的意義：** 如果你在 Max 方案，Dynamic Workflows 已自動開啟，下次跑複雜任務（如週報流程）時可以嘗試 `/effort ultracode` 看 Claude 是否自動拆分 subagent。
