1. 🚨 Anthropic June 15 計費變更 — Agent SDK 強制分池（直接影響 Claude Code 用法）

Anthropic 宣布自 2026-06-15 起，Agent SDK 相關用量從訂閱池分離，移至獨立月費 credit pool，以全額 API 費率計費，無 rollover。

  • Claude Agent SDK
  • claude -p（headless CLI）
  • Claude Code GitHub Actions
  • 第三方 Agent SDK 應用
不受影響（維持訂閱池）
  • Claude.ai web / desktop / mobile
  • Claude Code 在 terminal 互動式使用
  • Claude Cowork

重要細節： credit 按人頭計（不共池）、月底清零不滾入、用完後自動停擋（除非手動開啟 overflow billing）。原本訂閱隱性補貼了 15~30x API 費差，此次等同取消補貼。

在 6/15 前登入 Claude 帳號確認 credit 申領信件，並決定是否開啟 overflow billing。若每月 claude -p 用量超過 $20 等值，需開啟 overflow 或升級 Max 方案。
此變更從 IPO 動機角度可理解：在 IPO 前最大化 API 收益。claude -p 和 GitHub Actions 自動化任務直接受影響，本週確認額度後制定預算。

2. Claude Code：Dynamic Workflows 預設開啟 + Ultracode 新設定（2026-06-02~03）

兩個重要 Claude Code 行為更新同步落地：

Dynamic Workflows 預設狀態變更：

  • Max 和 Team 方案： Dynamic Workflows 現在預設開啟，無需任何設定
  • Enterprise： 預設關閉，管理員可在 Claude Code settings 啟用
  • Pro： 需手動在 /config → Dynamic Workflows 啟用

新設定：/effort ultracode — 結合 xhigh 推理力度 + 自動判斷何時啟動 Dynamic Workflows。Claude 自動評估任務複雜度，決定是否啟動大規模並行處理。使用方式：session 開始時輸入 /effort ultracode。

其他更新： Fallback models 支援、更廣的 deny-rule glob 規則、跨 session 訊息安全性強化、20+ 法律 MCP connector + 12 個 practice-area plugin（企業向）。

Managed Agents 升級： 現在可在自管 sandbox（Cloudflare、Daytona、Modal、Vercel）中運作，並連接私有 MCP server。

如果你在 Max 方案，Dynamic Workflows 已自動開啟。下次跑複雜任務時可嘗試 /effort ultracode，觀察 Claude 是否自動拆分 subagent 及 orchestration script 結構。

3. Anthropic 機密 IPO S-1 遞交 SEC（2026-06-01）

Anthropic 於 2026-06-01 向 SEC 遞交機密 Form S-1，正式啟動 IPO 程序：

  • 估值： ~$965B（距 Series H 融資 $65B 不到 1 週）
  • 月營收 run-rate： ~$47B（May 2026），前一年同期約 $10B，YoY 成長 4.7x
  • 競爭背景： OpenAI（$852B 估值）、SpaceX 同年預計三家 $1T 級 IPO
  • S-1 機密遞交代表財務細節尚未公開，尚未定股數或發行價
Anthropic 作為商業主體的透明度將大幅提升。June 15 billing change 可從 IPO 動機理解：在 IPO 前最大化 API 收益、讓財務模型更清晰。
