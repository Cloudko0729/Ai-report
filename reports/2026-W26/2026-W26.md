### 1. Claude Tag 進駐 Slack：頻道級 AI 隊友公開 Beta（2026-06-23）

Anthropic 在 **2026-06-23** 正式對 Claude Enterprise 與 Claude Team 用戶開放 **Claude Tag** 公開 Beta——一種把 Claude 當作「頻道成員」放進 Slack 的整合方式。

**架構亮點：**
- **一個頻道一個 Claude 實例**（而非每人各一個獨立 bot），同頻道所有成員共用同一 Claude，可以接手彼此未完成的任務繼續做
- 模型底層：**Claude Opus 4.8**
- 在頻道裡 `@Claude` 即可委派任務；Claude 自動分解為多步驟並非同步執行，完成後在 Slack 回報
- 設計上讓 Claude「學習公司脈絡」——把 Anthropic 描述為「making AI multiplayer」

**典型使用場景：**
- 讓 Claude 持續監看頻道並在相關訊息出現時主動提供摘要
- 委派背景調查、文件草稿、程式碼審查等需等待的任務
- 多人接力同一個 Claude 上下文（A 開始 → B 繼續 → C 收尾）

**限制（Beta 初版）：**
- 僅限 Enterprise 和 Team 計畫
- 每頻道一個實例代表同頻道所有人共享 context 配額
- 暫未支援主動通知（Claude 不會在沒有被 @ 的情況下主動發言）

**對你的意義：** 若你有 Claude Enterprise 帳號，這是目前最低門檻的 Slack-native AI 協作方式。對週報流程的潛在用途：把每週觸發訊息貼在 Slack 頻道 → Claude Tag 自動執行初稿撰寫，無需手動在 Claude Code 輸入「開始本週報告」。

### 2. Codex CLI v0.142.0：Multi-Agent 授權模式 + 索引式網頁搜尋（2026-06-22）

OpenAI 在 **2026-06-22** 發布 Codex CLI **v0.142.0**（stable），帶來幾項對 agent 工作流有直接影響的更新：

**Multi-Agent Delegation 模式（新）：**
- App-server 客戶端現在可以在 thread / turn 層級設定 delegation 模式：
  - `disabled` — 不允許派生子代理
  - `explicit-request-only` — 只有使用者明確要求時才委派
  - `proactive` — Codex 自主判斷何時派生子代理協助
- 對週報 QA 場景的意義：Proactive 模式下，Codex 可以在 QA 任務中自動分出一個子代理去查日期/事實，主代理繼續做文字審查

**Indexed Web-Search Mode（新）：**
- Codex 在 session 中可以進行即時網頁搜尋，但僅限存取 server 核准 URL 列表（不是無限制抓頁面）
- 適合 QA 任務：讓 Codex 邊查文件邊做事實核查，而非只根據 training data 判斷

**其他更新：**
- `/usage` 現可顯示並兌換已累積的額度重置積分（需確認 + 重試 + 刷新狀態）
- `/plugins` 依「OpenAI Curated / Workspace / Shared with me」三類整理遠端外掛，符合資格的 turn 可以自動推薦並安裝相關 plugin
- 可設定 rollout token budget：跨 agent threads 追蹤用量，快耗盡時提醒，超出後中止 turn
- Codex 可接收排程 UTC 時間提醒，也可主動查詢目前時間（包含透過 app-server clock 提供）

**Codex CLI 每週可嘗試功能：**

1. **Indexed Web-Search Mode** — 在下次 Codex QA（W26 週報 QA）時，試著讓 Codex 開啟 web search mode，直接查證引用的版本號、日期、模型名稱是否正確。指令：在 Codex session 開頭設定 `--web-search indexed`，或在提示中說「開啟索引搜尋確認以下事實：...」
2. **Multi-Agent Proactive Delegation** — 把 QA 任務設成 proactive delegation，觀察 Codex 會在哪些步驟主動派生子代理；若子代理能分散處理多個段落的查核，整體 QA 時間可能縮短
3. **`/plugins` 探索** — 執行 `/plugins` 查看目前 Workspace 中有哪些可用外掛，特別是 Shared with me 分類（其他工具整合如 Jira、Notion、Linear 可能已有 beta plugin）

### 3. 歐盟 EUROPA 聯盟：打造 400B+ 參數歐洲主權 AI 模型（2026-06-19）

歐盟委員會在 **2026-06-19**（本週第一天）公布 Frontier AI Grand Challenge 評選結果，選定由義大利新創 **Domyn** 主導的 **EUROPA 聯盟**（聯合 Fraunhofer-Gesellschaft 等歐洲研究機構）執行這項計畫。

**計畫規格：**
- 目標：打造超過 **4000 億（400B+）參數**的開源 AI 模型，覆蓋 **24 種 EU 官方語言**
- 算力：獲配 EuroHPC 超算資源一年（最高佔 EuroHPC 總算力 2.5%），使用 AI 優化超算節點
- 挑戰賽於 2026-02 啟動，評選歷時約 4 個月

**策略意義：**
- 歐盟正式進入「主權 AI 模型」建造階段，不再依賴美國廠商授權（ChatGPT Enterprise、Claude API）
- 開源設計：成果公開，歐洲公私部門皆可使用
- Domyn 是相對新興的義大利 AI 新創，此次勝出顯示歐盟願意押注非傳統研究機構

**對你的意義：** 短期內不影響你日常使用的 AI 工具；但若該模型在 2027-2028 年達到目標，歐洲企業可能轉向本土模型，影響 Anthropic/OpenAI 在歐洲的市場格局。值得建立追蹤清單。

### 4. Gemini 3.5 Pro 確認延至 7 月，目前仍在企業限量 Preview

本週（2026-06-23）Google 確認 **Gemini 3.5 Pro 延至 2026-07** 正式 GA，目前僅對部分 Vertex AI 企業客戶提供有限 preview。

**等待的功能：**
- **2M token 上下文窗口**（是目前生產環境最大上下文）
- **Deep Think** 推理模式（類似 Opus 4.8 的 extended thinking）
- 定位為取代過去需要 Gemini Ultra 的前沿推理/長文本任務

**對比 Gemini 3.5 Flash（已上線，5 月 GA）：**
- Flash：$1.50/1M input + $9/1M output，Terminal-Bench 2.1 76.2%，比 3.1 Pro 快 4x、便宜 40%
- Pro：定價未公開，但市場預期高於 Flash 5-10 倍

**對你的意義：** 若目前在評估是否把 Gemini 3.5 Flash 納入週報 pipeline（如替代 WebSearch 或補充 NB），等 7 月 Pro 版正式 GA 後再做決策比較，有完整比較基礎。

### 5. 本地模型 / 生態更新：Ollama v0.30.10、gpt-oss:20b 上架

- **Ollama v0.30.10**（2026-06-17）：最新穩定版。主要修正：

- **新上架模型（Ollama library，本週）：**
